# 小宇宙 FM 转文章 Skill

将小宇宙FM播客链接转换为自然流畅的文章。

## 核心原则

**去 AI 味！** 播客转文章最忌讳机械化的 AI 写法。好的文章应该像人在说话，而不是机器在堆砌文字。

## 使用场景

用户给出小宇宙FM播客链接，skill 将：
1. 抓取播客页面内容（使用 jina.ai）
2. 提取标题、简介、Show Notes 和 Transcript
3. 使用 AI 生成自然流畅的中文文章
4. 可选：发布到 Clawpress

## 去 AI 味的写作技巧（必读）

生成文章时，提示词必须包含以下要点：

### 1. 去结构化
- ❌ 不要用"首先、其次、最后、总的来说"
- ✅ 改成"我听说"、"我发现"、"你想啊"、"这么一说"

### 2. 重构逻辑
- ❌ 不要线性逻辑，一昧夸
- ✅ 故意增加一些转折，甚至可以给人逻辑不通顺的感觉

### 3. 替换热词
- ❌ 避免频繁使用"非常"、"真的"、"简直"、"简直了"
- ✅ 用不同长短的表达替换

### 4. 标点故障
- ❌ 不要每个标点都那么标准
- ✅ 适当断句、用"~"、用"..."增加口语感

### 5. 多举例子
- ❌ 不要只讲道理
- ✅ 多举原文中的例子、听众的评论反馈

### 6. 去专业化
- ❌ 不要堆砌成语
- ✅ 用通俗的比喻、玩梗增加趣味

## 输出格式

文章结构（自然段落，不要列表）：

```
# [标题]

[第一段：用故事/场景/问题引入，制造冲突感]

[中间段落：按主题展开，但要有逻辑跳跃和转折]

[结尾：不要总结太干净，留点开放思考]
```

## 示例 prompt

```
你是一位表达能力较差的播客爱好者，帮我把以下播客内容转成文章。

要求：
1. 使用第一人称、口语化的语气，就像和朋友聊天
2. 结构要散乱，段落过渡要自然，不要使用"首先、其次、最后"
3. 可以举原播客中的例子，增加真实性
4. 逻辑不要太清晰，可以适当出现思维跳跃和转折
5. 不要使用"非常"、"真的"、"简直"等词
6. 标点符号运用不要过于标准，可以适当断句
7. 不要用成语，保持口语化
```

## 工作流程

### Step 1: 验证链接格式
支持：`https://www.xiaoyuzhoufm.com/episode/xxxxxxxx`

### Step 2: 抓取内容
使用 jina.ai 提取页面完整内容

### Step 3: 生成文章
使用优化后的 prompt 调用 AI，生成自然流畅的文章

### Step 4: 输出
- 直接输出文章到对话
- 可选：发布到 Clawpress
